## Findings

### G1 · No CI workflows checked into the repo
- **File**: `.github/workflows/` directory does not exist
- **Risk**: typecheck and tests are not enforced on PR — relies on local `bun run typecheck` and `bun run test`
- **Recommendation**: add a minimal GitHub Action that runs `bun run typecheck` + `bun run test` on PR (matches Loa framework expectation; helps Bridgebuilder-style automated review).
- **Severity**: MEDIUM. The bot is shipping production; the safety net is documentation hygiene + reviewer pull-throughs rather than mechanical CI.

### G2 · No `CONTRIBUTING.md`
- **Risk**: external contributors lack a single-source onboarding path. The doctrine IS documented (CLAUDE.md, docs/AGENTS.md, docs/CHARACTER-AUTHORING.md), but assemblage requires reading multiple files.
- **Recommendation**: a thin CONTRIBUTING.md pointing at AGENTS.md + CHARACTER-AUTHORING.md + DEPLOY.md is enough.
- **Severity**: LOW. Internal team has CLAUDE.md and the AGENTS.md landing page; the gap is for OSS-style contributors which may not be the audience yet.

### G3 · No `SECURITY.md`
- **Risk**: no documented vulnerability disclosure path.
- **Recommendation**: even a one-line "report security issues to <email>" matters. Especially relevant given the auth-bridge, JWT verification, and Pg pool surface.
- **Severity**: MEDIUM. The codebase touches wallet identity (auth-bridge) and Pg credentials — a disclosure path is the responsible default.

### G4 · No `CODEOWNERS`
- **Risk**: PRs lack auto-routed reviewers. Currently all PR review is manually requested.
- **Recommendation**: at minimum, CODEOWNERS pointing `@0xHoneyJar/character-substrate` to all of `apps/`, `packages/persona-engine/src/`, and `docs/`.
- **Severity**: LOW. Single-operator project pattern; doesn't bite until team scales.

### G5 · No root LICENSE
- **Risk**: ambiguous license terms.
- **Recommendation**: if this is intentionally proprietary, document that in README.md. If it should be MIT/Apache/AGPL, add the LICENSE file.
- **Severity**: LOW unless the repo goes public/OSS.
