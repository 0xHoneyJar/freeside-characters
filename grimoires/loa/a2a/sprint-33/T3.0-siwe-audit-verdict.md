## Verdict: DO NOT FORK — build fresh (done in this sprint)

Sietch is **Express + EIP-191 personal_sign over a custom template**, not EIP-4361 SIWE, and not
Bun. It has strong *operational* security but does not meet the strict SIWE criteria. A literal
fork would transplant the wrong framework AND the gaps. We built fresh in Bun + @noble, porting
sietch's validated ops patterns.

## The 5 criteria

| # | Criterion | sietch | verdict |
|---|---|---|---|
| 1 | Server-side single-use nonce | ✅ `NonceManager.isValid` checks `used`; `markUsed` consumes | PRESENT — pattern ported |
| 2 | `domain` == static VERIFY_ORIGIN | ⚠️ enforced as Origin/Referer **CSRF** check (`validateOrigin`), NOT as a cryptographic `domain` field in the signed message | PARTIAL gap |
| 3 | `chainId` pinned | ❌ EIP-191 personal_sign is chain-agnostic; no chainId in message | GAP |
| 4 | `expirationTime` ≤ 5m | ⚠️ nonce TTL default **15m**; no SIWE `expirationTime` field | GAP (exceeds 5m) |
| 5 | URI/authority bound (EIP-4361) | ❌ custom template (community/wallet/discord/nonce/timestamp), not EIP-4361 structure | GAP |

Gate rule (sprint plan): "a gap in {single-use nonce · domain-pin · chainId-pin · ≤5m expiry}
BLOCKS the fork." Criteria 2/3/4 are gaps → **fork blocked**.

## What we built instead (meets all 5)

- **EIP-4361 message** — `persona-engine/src/onboarding/siwe.ts` `buildSiweMessage` (domain, address,
  statement, URI, Version 1, Chain ID, Nonce, Issued At, Expiration Time). Criteria 2/3/5. ✅
- **chainId pin** — `verifySiweSignature(..., pinnedChainId)` rejects mismatch. Criterion 3. ✅
- **≤5m single-use nonce** — `verify-session.ts` `issueSiweNonce` (5m TTL) + `claimSiweNonce`
  (atomic O_EXCL claim, one winner). Criteria 1/4. ✅
- **domain pin / forked-host replay (RT-1)** — server reconstructs the canonical message from STATIC
  config (domain, chainId, uri) + session nonce + claimed address; `recover(reconstructed, sig) ==
  claimedAddress`. A signature minted on a forked host (different domain) recovers a different
  address → reject. Tested (`siwe.test.ts` RT-1). ✅

## Ops patterns ported from sietch (validated there, kept here)

- Origin/Referer CSRF check (`verify-routes.ts` `originOk`, RT-7/ATK-006).
- Single-use nonce semantics (NonceManager → `claimSiweNonce`).
- No-secret-in-client posture; `Referrer-Policy: no-referrer` + `no-store` + `X-Frame-Options: DENY`.

## NOT ported (deliberately)

- Express middleware, rate-limiter (`express-rate-limit`), session-UUID model — replaced by the
  opaque handoff-token (C3) + Bun routes.
- Constant-time response padding (`MIN_RESPONSE_TIME_MS`) — candidate v1.1 hardening; the verify
  surface's timing channel is low-value (no per-user secret compared in the hot path), tracked not v1.
