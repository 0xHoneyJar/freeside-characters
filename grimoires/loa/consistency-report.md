## Conflicts / Low-Cost Improvement Opportunities

### C1 · Mixed casing in `apps/character-mongolian/character.json` field naming
- Evidence: most fields camelCase (`displayName`, `personaFile`, `exemplarsDir`, `emojiAffinity`, `webhookAvatarUrl`); but quest substrate fields snake_case (`quest_substrate`, `rubric_pointer`, `mention_allowed_channels`, `slash_allowed_channels`, `submission_style_override`, `positive_friction_delay_ms_override`, `voice_cadence`).
- Severity: LOW
- Rationale: snake_case appears to mirror a wire-contract shape from an external system (likely freeside-quests/engine). Mixing in JSON is annotational, not structural — interfaces remain typed.
- Recommendation: document the convention in `docs/CHARACTER-AUTHORING.md` ("character.json uses camelCase for substrate fields, snake_case for quest_substrate fields to mirror the freeside-quests wire contract"). Or normalize when that contract lands.

### C2 · `slash_commands` field is snake_case in the public type
- Evidence: `packages/persona-engine/src/types.ts:96` — `slash_commands?: SlashCommandSpec[]`; `mcps?: string[]` line 118; `tool_invocation_style?: string` line 141.
- Severity: LOW
- Rationale: comment block above does not explain the snake_case choice (most TS surface uses camelCase). The pattern leaks via Discord JSON conventions (Discord application command shapes use snake_case).
- Recommendation: add a doc comment explaining "mirrors Discord JSON shape" OR normalize to `slashCommands`. Compile-time-only change with a one-shot grep update.

### C3 · `mcps` vs `MCPs` capitalization in prose
- Evidence: code uses `mcps?: string[]` (line 118 types.ts); CLAUDE.md uses both `MCP` and `mcps` mixed; some docs say `mcp__` prefix for runtime tool names.
- Severity: TRIVIAL
- Recommendation: settle on `mcps` field + `MCP servers` prose. Already nearly there.

### C4 · "Character" terminology drift in shell-bot context
- Evidence: CLAUDE.md uses "shell-bot static templates" in commit messages and `Loa identity strip`; types.ts uses "shell account"; webhook.ts says "shell"; README.md "one shell, many speakers" doctrine codified.
- Severity: NONE — this is consistent, just noting it because it's a domain-specific term that's clearly defined.

### C5 · Cycle/Sprint provenance in code comments
- Pattern: most files cite cycle + sprint + PR + author (e.g. "cycle-B sprint-1 review fix · PR #53 · cross-reviewer flatline").
- Severity: STRENGTH (not a conflict).
- Note: this is a notable hygiene strength worth preserving. Loa session continuity benefits from this density.

## Breaking-Change Watch (flag, don't fix)

- File: `packages/persona-engine/src/score/types.ts:117-167`
- Pattern: `RawStats` carries both v1 (`total_events`, `active_wallets`) and v2 (`top_event_count`, `top_wallet_count`, `window_event_count`, `window_wallet_count`) field shapes. Helper functions (`getWindowEventCount`, `getWindowWalletCount`, etc.) read regardless of version.
